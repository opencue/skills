# MCP snapshots — moved

MCP server snapshots have been deduped and now live in their own repo:

→ **[recodeee/mcps](https://github.com/recodeee/mcps)** — `configs/`

Sources covered there (regenerated by the systemd timer + Stop hook every sync):

- `~/.codex/config.toml` — Codex CLI MCP servers
- `~/.claude/settings.json` `mcpServers` — Claude Code static settings
- `~/.claude.json` top-level `mcpServers` — what `claude mcp add` writes
- `~/.claude.json` `projects[*].mcpServers` — per-project MCPs
- `~/.config/Code/User/mcp.json` — VS Code MCP

Plus the plugin-bundled MCPs at [recodeee/mcps `plugins/`](https://github.com/recodeee/mcps/tree/main/plugins).

This `mcps/` directory inside `recodeee/skills` is kept only for the
skill-to-MCP mapping registry (`mcp-skill-map.json`, `mcp-skill-rules.json`)
when those files are generated by the skill workflow. Snapshot files
(`claude-*.json`, `codex-*.json`) are not regenerated here.

History note: `scripts/sync-mcps.sh` was removed on 2026-05-09. The
canonical snapshot script is `recodeee/mcps/scripts/snapshot-mcps.py`.
